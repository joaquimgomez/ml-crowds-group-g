PedestrianSpeedInAreaProcessorUsingAgentVelocity

Log for each pedestrian the speed within a measurement area. 

+--------------------------------------------------+
|                                                  |
|        M1-------------+  M2-------------+        |
| +-+    |              |  |              |    +-+ |
| |S|    |      P1      |  |      P2      |    |T| |
| +-+    |              |  |              |    +-+ |
|        +--------------+  +--------------+        |
|                                                  |
+--------------------------------------------------+

- S: source
- T: target
- Pi: pedestrian i
- Mj: measurement j
Note: If two measurement areas M1 and M2 are disjoint and a pedestrian P1 is located within M1, M2 should log a speed of -1 for P1. 
| timeStep | pedId | ... | M1  | M2  |
|----------|-------|-----|-----|-----|
|1         | 1     |     | 0.5 | -1  |
|1         | 2     |     | -1  | 0.6 |
Watch out: This processor uses the PedestrianVelocityDefaultProcessor internally instead of the PedestrianTrajectoryProcessor which is used by PedestrianSpeedInAreaProcessorUsingAgentTrajectory.

Field: pedestrianVelocityDefaultProcessor
