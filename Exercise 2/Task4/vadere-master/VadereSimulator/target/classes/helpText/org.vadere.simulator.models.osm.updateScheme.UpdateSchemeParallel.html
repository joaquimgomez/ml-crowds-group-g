UpdateSchemeParallel

The implementation of the parallel update scheme described in 'How update schemes influence crowd simulations' i.e. seitz-2014b. The implementation is not based on the time credit concept but on event times similar to the event driven update scheme. The difference is that all events which starts within a time span of currentTimeInSec - timeStepInSec and currentTimeInSec will be performed in parallel on the bases of the situation (i.e. agents position) at currentTimeInSec - timeStepInSec.
